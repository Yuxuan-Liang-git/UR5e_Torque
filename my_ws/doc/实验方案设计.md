# NMPC 基于 VLA 输出轨迹的最小实机验证方案

## 1. 实验目的

使用 `config/trajectory/final_exec_160704.csv` 中保存的 VLA 输出关节角度信息作为测试轨迹，验证 NMPC 相较于普通 PD 控制在真实 VLA 轨迹跟踪中的作用。

本阶段只做最小测试单元：

```text
同一条 VLA 输出轨迹
同一个被测关节
PD 与 NMPC 两种控制器
比较跟踪误差、加速度抖动、jerk 和力矩变化
```

暂不进行 NMPC 参数调节，不测试多关节协同，不测试异常点、延迟、丢帧和复杂任务场景。

## 2. 测试数据

测试数据使用：

```text
config/trajectory/final_exec_160704.csv
```

该文件视为 VLA 输出的关节角度序列。

第一阶段只选取其中一个关节作为参考轨迹，例如：

```text
joint2
```

其余关节保持初始姿态不动。

## 3. 实验对照组

| 组别 | 控制器 | 参考轨迹 |
|---|---|---|
| A | PD | `final_exec_160704.csv` 中单个关节轨迹 |
| B | NMPC | 同一条单个关节轨迹 |

两组实验必须保持一致：

1. 同一个被测关节。
2. 同一段 CSV 轨迹。
3. 同一初始姿态。
4. 同一控制频率。
5. 同一力矩限制。
6. 同一摩擦补偿设置。
7. 同一运行时长。

## 4. 参考轨迹构造

假设选择 `joint2` 作为被测关节，则：

```text
q_ref_joint2(t) = final_exec_160704.csv 中 joint2 的角度序列
```

其他关节保持初始值：

```text
q_ref_joint_i(t) = q_start_joint_i, i != 2
```

因此整体参考轨迹为：

```text
q_ref(t) = [
  q_start_0,
  q_start_1,
  q_vla_2(t),
  q_start_3,
  q_start_4,
  q_start_5
]
```

如果 `final_exec_160704.csv` 的采样周期与控制周期不同，先按当前程序已有读取方式执行，不额外加入插值滤波；本阶段重点是保持 PD 和 NMPC 输入完全一致。

## 5. 实验步骤

### 5.1 PD 实验

1. 机械臂移动到安全初始姿态 `q_start`。
2. 读取 `final_exec_160704.csv`。
3. 只取被测关节轨迹，其余关节参考保持 `q_start`。
4. 启动数据记录。
5. 使用 PD 控制器跟踪该单关节 VLA 轨迹。
6. 轨迹执行结束后停止控制并保存数据。
7. 数据文件命名为：

```text
PD_final_exec_joint2_trial01.csv
```

### 5.2 NMPC 实验

1. 机械臂重新移动到同一安全初始姿态 `q_start`。
2. 读取同一个 `final_exec_160704.csv`。
3. 只取同一个被测关节轨迹，其余关节参考保持 `q_start`。
4. 启动数据记录。
5. 使用当前已部署 NMPC 控制器跟踪该单关节 VLA 轨迹。
6. 轨迹执行结束后停止控制并保存数据。
7. 数据文件命名为：

```text
NMPC_final_exec_joint2_trial01.csv
```

如果时间允许，每组重复 3 次：

```text
trial01
trial02
trial03
```

若时间紧，先各跑 1 次，确认流程和评价脚本跑通。

## 6. 需要记录的数据

最小记录字段如下：

| 字段 | 说明 |
|---|---|
| `t` | 时间 |
| `q_ref_j` | 被测关节参考角度 |
| `q_j` | 被测关节实际角度 |
| `dq_j` | 被测关节实际速度 |
| `tau_j` | 被测关节最终发送力矩 |
| `control_dt_ms` | 控制周期 |
| `solve_time_ms` | NMPC 求解时间，PD 组可填 0 |

如果当前日志已经保存 6 个关节的完整数据，可以直接使用完整日志，只需在分析时提取被测关节。

## 7. 评价指标

只计算以下基础指标。

### 7.1 跟踪误差

```text
e_q[k] = q_ref_j[k] - q_j[k]
RMSE_q = sqrt(mean(e_q^2))
Max_Error_q = max(abs(e_q))
```

### 7.2 加速度抖动

由关节速度差分得到加速度：

```text
ddq[k] = (dq_j[k] - dq_j[k-1]) / dt
RMS_ddq = sqrt(mean(ddq^2))
```

### 7.3 jerk

由加速度差分得到 jerk：

```text
jerk[k] = (ddq[k] - ddq[k-1]) / dt
RMS_jerk = sqrt(mean(jerk^2))
Jerk_integral = sum(jerk^2) * dt
```

### 7.4 力矩变化

```text
delta_tau[k] = tau_j[k] - tau_j[k-1]
dTau_RMS = sqrt(mean(delta_tau^2))
Tau_peak = max(abs(tau_j))
```

### 7.5 实时性

```text
Control_frequency_mean
Control_frequency_min
Mean_solve_time_ms
Max_solve_time_ms
Solver_failure_count
```

## 10. 判断标准

NMPC 的作用不一定表现为跟踪误差最小，而是表现为在可接受跟踪误差下让执行更平滑。

若满足：

```text
NMPC 的 RMS_jerk < PD
NMPC 的 dTau_RMS < PD
NMPC 的 Tau_peak 不明显高于 PD
NMPC 的 RMSE_q 没有明显恶化
```

则可认为：

```text
NMPC 相较于普通 PD，在跟踪 VLA 输出轨迹时降低了关节运动冲击和力矩突变，具备轨迹平滑与优化控制效果。
```

如果 NMPC 的平滑性指标暂时没有优于 PD，不直接否定 NMPC，只说明当前 NMPC 参数、权重或轨迹输入方式还需要后续调整。

## 11. 停止条件

实验中出现以下情况应立即停止：

1. 机械臂明显振动。
2. 关节运动方向异常。
3. 力矩连续限幅。
4. 控制频率低于 100 Hz。
5. NMPC 连续求解失败。
6. 机械臂接近工作空间边界。

## 12. 本阶段完成标准

完成以下内容即可认为本阶段最小验证跑通：

1. 使用 `final_exec_160704.csv` 生成单关节参考轨迹。
2. 完成一次 PD 跟踪实验。
3. 完成一次 NMPC 跟踪实验。
4. 保存两份数据。
5. 计算跟踪误差、加速度抖动、jerk 和力矩变化指标。
6. 画出关节角、加速度/jerk、力矩三类对比图。
7. 得到一句初步结论：

```text
在 final_exec_160704.csv 单关节 VLA 轨迹跟踪测试中，NMPC 相较于 PD 是否降低了 jerk 和力矩变化。
```
